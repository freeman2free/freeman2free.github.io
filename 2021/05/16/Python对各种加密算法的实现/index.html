发表于 2021-05-16 更新于 2021-05-20 分类于 Study

认实并理解各种加密算法的原理是学好逆向的关键

一. BASE64编码

base64是基于64个可打印字符来表示二进制数据的方法，严格来说他算是一种伪加密方式，因为他没有密钥注入算法，只要拿到密文进行base64解密就能得到相应的明文。

base64的识别也很简单，由于其特性，要编码的二进制数据如果不是3的倍数，最后会剩下1个或2个字节，Base64用\x00字节在末尾补足后，再在编码的末尾加上1个或2个=号，表示补了多少字节。根据这个特性，一般密文结尾处如果是’=’结尾的，可以先判断是base64编码而成。

二. 哈希散列算法

就是一种将任意长度的消息压缩到某一固定长度的消息摘要的函数，摘要算法不能被称为加密算法，因为它是单向操作的，即只可将明文变密文，基于这种单向操作的特性，摘要算法一般被用于确保信息传输完整一致。常用的有以下几种：

​ MD5-密码散列函数，生成的值是唯一的,且不能解密只能加密不可逆,如果被修改该值就会变化用于校验文件一致性,还可以加个salt盐值起到混淆作用增加安全性。流程大概为：处理原文 - 设置初始值 - 循环加工 - 拼接结果(MD5的信息摘要位数为:128bit)

​ SHA-安全散列算法,是一个密码散列函数家族,能计算出一个数字消息所对应到的，长度固定的字符串（消息摘要）的算法,包含五个算法分别是SHA-1(信息摘要位数160bit)、SHA-224(224bit)、SHA-256(256bit)、SHA-384(384bit)，和SHA-512(512bit)

​ SM3-国密算法的一种，消息摘要，算法已公开。校验结果为256bit，密钥长度和分组长度均为128bit

​ 补充:HMAC是密钥相关的哈希运算消息认证码（Hash-based Message Authentication Code）的缩写，一种基于Hash函数和密钥进行消息认证的方法，它可以与任何迭代散列函数捆绑使用。

三. 对称加密算法

该类算法常见的有以下几种:

DES-分组加密算法：将明文拆分为固定长度的组，每组都采用同一密钥和算法进行加密，最后输出固定长度的密文，但是由于des的密钥每隔八位设置为奇偶校验位，因此密钥长度实际为56位，与明文的64位分组不相符。流程大致为:初始置换 - 生成子密钥 - 迭代更新 - 逆置换

3DES-在DES基础上再经过3次加密

AES-高级加密标准：该算法有五种工作模式：ECB电码本模式 CBC密码分组链接模式 CTR计算器模式 CFB密码反馈模式 OFB输出反馈模式

SM1-国密算法：强度与AES相当，密钥长度和分组长度均为128位，该算法不公开，调用该算法时，需要通过加密芯片的接口进行调用

SM4-无线局域网标准的分组数据算：密钥长度和分组长度均为128位

不过无论哪种算法，其核心都是密钥，只要得到密钥就等于得到数据，因为该类算法使用的密钥也可以用来解密只有一把钥匙(即公钥私钥为同一把)，可逆

四. 非对称加密算法

该类算法加密和解密的钥匙是不同的，一个公钥用于加密一个私钥用于解密有两把密钥各有作用，主要包括以下几种:

RSA-其加解密原理流程为：
公钥=(E, N) 私钥=(D, N) 找出质数 P,Q；
计算公共模数 N=P * Q；
欧拉函数 φ(N)=(P-1)(Q-1)；
计算公钥E 1 < E < φ(N) ；
计算私钥D E*D%φ(N)=1 ；
加密 C(密文)=M(明文) ** E mod N ；
解密 M(明文)=C(密文) ** D mod N

（识别方式：RSA公钥加密会进行类似MD5加盐的操作，所以相同的明文，用相同的公钥进行RSA加密会生成不同的密文）

DSA-该算法不单单只有公钥、私钥，还有数字签名。私钥加密生成数字签名，公钥验证数据及签 名。如果数据和签名不匹配则认为验证失败，数字签名的作用就是校验数据在传输过程中不被修改(数字签名，是单向加密的升级。)

(但是DSA算法仅产生数字证书，信息仅能用于验证，不适合进行加密通信，所以HTTPS不会使用这个算法，一般不用于加密算法中)

加密流程为:

​ 发送方：产生一个随机数，将信息、随机数、私钥传入到算法中；

​ 发送方：算法计算一个签名串，将签名串和原始数据一起传输给接收方；

​ 接收方：将接收到的信息、签名串发送给验证函数，验证函数使用发送方公钥来比较验证签名信息；

ECC-相比RSA，ECC优势是可以使用更短的密钥，来实现与RSA相当或更高的安全性。

加解密算法原理：

​ 设私钥、公钥分别为d、Q，即Q = dG，其中G为基点，椭圆曲线上的已知G和dG，求d是非常困难的，也就是说已知公钥和基点，想要算出私钥是非常困难的。

​ 公钥加密：选择随机数r，将消息M生成密文C，该密文是一个点对，C = {rG, M+rQ}，其中Q为公钥。

​ 私钥解密：M + rQ - d(rG) = M + r(dG) - d(rG) = M，其中d、Q分别为私钥、公钥。

SM2-国密算法，由于基于ECC，因此其签名速度与秘钥生成速度都快于RSA，算法已公开，采用的是ECC 256位的一种，安全强度比RSA 2048位高，但运算速度快于RSA
